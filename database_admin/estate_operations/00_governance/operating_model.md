# DBA Operating Model (100+ DBA Team)

## Team topology
- Platform DBAs (engine specialists)
- Reliability DBAs (SRE + incident command)
- Automation DBAs (tooling/orchestration)
- Compliance DBAs (security + audit)

## Core cadences
- Daily operations review
- Weekly risk/perf review
- Monthly architecture + capacity review
- Quarterly DR and restore game days

## Mandatory controls
- Peer review for DDL/parameter changes
- Change windows with rollback plans
- Evidence capture for all critical operations
